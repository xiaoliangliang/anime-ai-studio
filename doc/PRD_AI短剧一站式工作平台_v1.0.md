## 〇、MVP 范围说明

### 0.1 MVP 约束
1. **单机本地优先**：单人场景，数据存储在浏览器本地（IndexedDB）
2. **仅 Web 桌面端**：暂不考虑移动端/平板端适配
3. **暂不做项目导出/成片导出与批量下载**：项目导出、合成导出、图片/视频批量下载等功能延后（**MVP 允许单资源下载：单图/单视频**）
4. **暂不做团队协作**：不考虑多人分工与协作
5. **导演阶段精简**：仅"逐镜头生成+预览"，不做音频/字幕/时间线编辑
6. **统计面板精简**：仅保留最小统计（生成成功/失败计数）

> **关于下载/导出功能的说明**：
> - **MVP 不含**：项目导出、批量下载（ZIP）、合成成片导出等
> - **MVP 包含**：单张图片下载、单个镜头视频下载（从资源详情/预览入口触发）
> - **备注**：仍可使用浏览器右键“另存为”（兜底）
> - **理由**：打包/格式转换/合成链路复杂，统一放到 MVP 后

### 0.2 MVP 验收路径
1. 首页：输入一句话 → 创建项目 → 进入编剧阶段
2. 编剧：生成剧本（至少第1集）→ 用户可编辑 → 标记编剧阶段完成
3. 分镜：引用第1集剧本 → 生成分镜表 → 用户可编辑 → 标记分镜阶段完成
4. 图像设计：生成角色/场景参考提示词 → 生成关键帧提示词
5. 美工：批量生成关键帧图（至少覆盖第1集全部镜头）
6. 导演：逐镜头生成视频片段（至少3个镜头成功）→ 可预览播放

### 0.3 技术架构概要
- **前端框架**: React 18 + TypeScript + Vite
- **无限画布**: tldraw 2.0（统一选型，不使用 Fabric/Konva 等其他方案）
- **本地存储**: IndexedDB（元数据 + 媒体资产本地缓存）
- **图片托管**: imgbb（**MVP必做**，云端永久链接供AI API引用）
- **AI服务**: Pollinations.AI（Chat/Image/Video API）
- **系统提示词**: 项目内版本化 `src/prompts/*.json`

> **存储策略**：图片采用"云托管(imgbb) + 本地缓存(IndexedDB)"双写机制。
> AI API需要外部可访问的URL，因此imgbb云托管为MVP必做项。

### 0.4 交互方式说明
采用**混合式交互**：AI 用对话方式引导创作，但在关键节点提供按钮操作。

| 场景 | 交互方式 | 示例 |
|------|----------|------|
| 参数收集 | 对话引导 | "请告诉我故事类型和集数" |
| 确认节点 | 按钮操作 | [确认大纲] [重新生成] |
| 批量生成 | 按钮触发 | [开始生成] [暂停] [继续] |
| 单项操作 | 按钮直接操作 | [重新生成此镜头] [编辑] |

### 0.5 过期机制（Stale Mechanism）
当上游内容被修改时，下游关联内容会被标记为"过期"，需要重新生成。

**追踪粒度**：按集/场景/镜头级别追踪变更，只标记受影响的下游。

**变更传播规则**：
| 修改内容 | 影响范围 | 处理方式 |
|----------|----------|----------|
| 剧本某集内容 | 该集的分镜/提示词/图片/视频 | 标记为过期 |
| 分镜某镜头 | 该镜头的提示词/关键帧/视频 | 标记为过期 |
| 提示词 | 关联的图片/视频 | 标记为过期 |

**UI 表达**：
- 过期内容显示 ⚠️ 徽标
- 提供 [重新生成] 按钮
- 支持批量重新生成受影响内容

**孤儿数据策略**：允许下游数据独立存在，即使上游被删除，下游数据仍然保留（不自动级联删除）。

### 0.6 关键帧拆分规则
根据镜头时长自动确定关键帧数量：

| 镜头时长 | 关键帧数量 |
|----------|------------|
| ≤3秒 | 1帧 |
| 3-6秒 | 2帧 |
| >6秒 | 3帧 |

## 一、产品概述

### 1.1 产品定位
AI短剧一站式工作平台是一个集成AI能力的短剧创作工具，覆盖从剧本创作到视频生成的完整工作流程。用户通过与AI对话协作，逐步完成编剧、分镜、图像设计、美工出图、视频合成等环节。

### 1.2 目标用户
- 短剧创作者/编剧
- 自媒体内容创作者
- 短视频制作团队
- AI视频爱好者

### 1.3 核心价值
- **一站式**: 从创意到成片，无需切换多个工具
- **AI驱动**: 每个阶段由专业AI助手辅助完成
- **可视化**: 无限画布承载全流程产出，随时查看修改

- 画布状态 (tldraw JSON快照)
- 对话历史 (每个阶段独立)

## 五、技术架构 (概要)

> **注意**：详细技术方案请参阅 `技术方案_AI短剧一站式工作平台.md`。本章仅作概要，以技术方案文档为准。

### 5.1 前端技术栈
- **框架**: React 18 + TypeScript + Vite
- **无限画布**: tldraw 2.0
- **状态管理**: React Context（MVP）
- **样式**: CSS Modules（或普通CSS）

### 5.2 存储方案（MVP）
- **IndexedDB**: 本地缓存项目元数据和媒体资产（图片/视频 Blob）
- **imgbb**: **MVP必做**，云端图片托管，供AI API引用外部可访问的URL
- **画布快照**: tldraw JSON 存储在 IndexedDB
- **双写策略**: 图片同时写入imgbb云端 + IndexedDB本地缓存

### 5.3 AI能力集成（MVP）
- **统一接口**: Pollinations.AI（通过本地代理 server.js 访问）
  - 文生文: Chat Completions（`/api/chat` → `text.pollinations.ai/v1/chat/completions`）
  - 文生图/图生图: `/api/txt2img` / `/api/img2img` → `gen.pollinations.ai/image`
  - 图生视频: `/api/img2video` → `gen.pollinations.ai/image`（seedance 模型，返回 `video/*`）

### 5.4 核心数据流

```
用户输入 → AI编剧 → 剧本数据 → AI分镜师 → 分镜数据 → AI图像设计师 → 提示词
                                                                    ↓
视频输出 ← AI导演 ← 视频片段 ← AI美工 ← 图片资源 ←────────────────────┘
```

## 六、非功能需求

### 6.1 性能需求
- 页面首屏加载 < 3秒
- AI响应延迟 < 5秒 (文生文)
- 图片生成 < 30秒/张
- 视频生成 < 2分钟/10秒片段

### 6.2 兼容性需求
- 浏览器: Chrome 90+, Firefox 90+, Safari 14+, Edge 90+
- 分辨率: 最低 1280x720, 推荐 1920x1080

### 6.3 数据安全
- 用户数据加密存储
- 生成内容版权归用户所有
- 支持项目导出备份（⏸️ MVP后）
