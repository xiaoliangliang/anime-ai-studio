# QA Checklist: 美工阶段图片生成优化

本文档提供美工阶段图片生成功能的回归测试用例清单。

## 测试环境准备

1. 确保项目已完成图像设计阶段（有 `imageDesigner` 数据）
2. 确保 API 密钥已配置（`RUNCOMFY_API_TOKEN`, `IMGBB_API_KEY`）
3. 打开浏览器开发者工具以查看日志

---

## 1. 并发生成测试

### 1.1 逐张展示

**测试步骤：**
1. 进入美工阶段
2. 点击"生成参考图"按钮
3. 观察画布区域

**预期结果：**
- [ ] 每张图片生成完成后立即显示在画布上
- [ ] 不需要等待所有图片完成才显示
- [ ] 图片按稳定顺序排列（不会因完成顺序不同而改变位置）

### 1.2 进度统计

**测试步骤：**
1. 开始生成图片
2. 观察进度显示区域

**预期结果：**
- [ ] 显示当前阶段（角色/场景/关键帧）
- [ ] 显示 "X/Y completed" 格式的进度
- [ ] 显示当前正在生成的图片名称
- [ ] 进度数字准确反映实际完成数量

### 1.3 Blocked 时序（含 in-flight 收尾）

**测试步骤：**
1. 开始生成图片
2. 等待某张图片失败（或模拟网络错误）
3. 观察状态变化

**预期结果：**
- [ ] 失败后停止调度新任务
- [ ] 已发出的请求（in-flight）允许自然完成
- [ ] 所有 in-flight 请求完成后进入 `blocked` 状态
- [ ] UI 显示失败原因和"重新生成"按钮

---

## 2. 暂停/继续测试

### 2.1 暂停功能

**测试步骤：**
1. 开始生成图片
2. 在生成过程中点击"暂停"按钮

**预期结果：**
- [ ] 立即停止调度新任务
- [ ] 显示 "pausing" 状态（如有 in-flight 请求）
- [ ] in-flight 请求完成后进入 "paused" 状态
- [ ] 已完成的图片保持不变

### 2.2 继续功能

**测试步骤：**
1. 在暂停状态下点击"继续"按钮

**预期结果：**
- [ ] 从下一个 pending 图片继续生成
- [ ] 不重新生成已完成的图片
- [ ] 进度从暂停时的位置继续

---

## 3. 重试功能测试

### 3.1 retryFailed(includePending=true)

**测试步骤：**
1. 生成过程中有图片失败
2. 点击"重新生成"按钮

**预期结果：**
- [ ] 只重新生成 `failed` 状态的图片
- [ ] 继续生成 `pending` 状态的图片
- [ ] `completed` 状态的图片保持不变（不重发请求）
- [ ] 成功图片的 `assetId` 和 `imageUrl` 不变

### 3.2 成功图片保留验证

**测试步骤：**
1. 记录重试前成功图片的 ID 和 URL
2. 执行重试
3. 对比重试后的图片数据

**预期结果：**
- [ ] 所有成功图片的 `id` 保持不变
- [ ] 所有成功图片的 `assetId` 保持不变
- [ ] 所有成功图片的 `imageUrl` 保持不变

---

## 4. 关键帧依赖检查

### 4.1 参考图不全时阻止

**测试步骤：**
1. 只生成部分角色/场景参考图
2. 尝试生成关键帧

**预期结果：**
- [ ] 关键帧生成被阻止
- [ ] 显示明确的错误提示（列出缺失/失败的参考图）
- [ ] 提示用户先完成参考图生成

### 4.2 参考图补齐后放开

**测试步骤：**
1. 完成所有角色/场景参考图生成
2. 再次尝试生成关键帧

**预期结果：**
- [ ] 关键帧生成正常启动
- [ ] 使用已完成的参考图作为输入

---

## 5. 幂等性和防抖测试

### 5.1 多次点击 start

**测试步骤：**
1. 快速多次点击"开始生成"按钮

**预期结果：**
- [ ] 只启动一次生成流程
- [ ] 不会创建多个并行的生成任务

### 5.2 多次点击 pause

**测试步骤：**
1. 在生成过程中快速多次点击"暂停"按钮

**预期结果：**
- [ ] 正常进入暂停状态
- [ ] 不会产生异常或错误

### 5.3 多次点击 resume

**测试步骤：**
1. 在暂停状态下快速多次点击"继续"按钮

**预期结果：**
- [ ] 正常恢复生成
- [ ] 不会启动多个并行的生成任务

### 5.4 多次点击 retry

**测试步骤：**
1. 在 blocked 状态下快速多次点击"重新生成"按钮

**预期结果：**
- [ ] 只执行一次重试
- [ ] 不会重复生成同一张图片

---

## 6. 状态持久化测试

### 6.1 刷新页面后恢复

**测试步骤：**
1. 生成部分图片后刷新页面
2. 重新进入美工阶段

**预期结果：**
- [ ] 已完成的图片正确显示
- [ ] 进度正确恢复
- [ ] 可以继续生成剩余图片

### 6.2 关闭浏览器后恢复

**测试步骤：**
1. 生成部分图片后关闭浏览器
2. 重新打开应用并进入项目

**预期结果：**
- [ ] 已完成的图片从 IndexedDB 正确恢复
- [ ] 失败的图片状态正确恢复
- [ ] 可以使用"重新生成"继续

---

## 7. 单张图片重生测试

### 7.1 单张失败图片重生

**测试步骤：**
1. 找到一张失败的图片
2. 点击该图片上的"重新生成"按钮

**预期结果：**
- [ ] 只重新生成该图片
- [ ] 其他图片状态不变
- [ ] 成功后更新该图片的显示

### 7.2 隔离性验证

**测试步骤：**
1. 记录所有图片的状态
2. 重新生成单张图片
3. 对比其他图片的状态

**预期结果：**
- [ ] 其他图片的 `status` 不变
- [ ] 其他图片的 `assetId` 不变
- [ ] 其他图片的 `imageUrl` 不变

---

## 8. 边界情况测试

### 8.1 空项目

**测试步骤：**
1. 创建新项目但不完成图像设计阶段
2. 尝试进入美工阶段

**预期结果：**
- [ ] 显示提示"请先完成图像设计阶段"
- [ ] 不会崩溃或显示错误

### 8.2 网络断开

**测试步骤：**
1. 开始生成图片
2. 断开网络连接

**预期结果：**
- [ ] 正在进行的请求失败
- [ ] 进入 blocked 状态
- [ ] 显示网络错误信息
- [ ] 恢复网络后可以重试

### 8.3 API 限流 (429)

**测试步骤：**
1. 快速发起大量请求触发限流

**预期结果：**
- [ ] 正确处理 429 错误
- [ ] 显示限流提示
- [ ] 可以稍后重试

## 测试工具

### 浏览器控制台测试

可以在浏览器控制台运行以下测试：

```javascript
// 快速测试（不调用 API）
const { runQuickTests } = window.generationControllerE2ETests;
await runQuickTests('your-project-id');

// 完整测试（会调用 API）
const { runAllE2ETests } = window.generationControllerE2ETests;
await runAllE2ETests('your-project-id');
```

### 单独测试函数

```javascript
const tests = window.generationControllerE2ETests;
const ctx = tests.createTestContext('your-project-id');

// 测试状态恢复
await tests.testStateRestorationFlow(ctx);

// 测试关键帧依赖检查
await tests.testKeyframeDependencyCheck(ctx);

// 测试完整生成流程（会调用 API）
await tests.testCompleteGenerationFlow(ctx);

// 测试暂停/继续流程（会调用 API）
await tests.testPauseResumeFlow(ctx);

// 测试失败重试流程（会调用 API）
await tests.testRetryFailedFlow(ctx);
```
