## 第4页：技术架构

### 视觉概念
技术栈图标以模块化方式排列，数据流用箭头连接

### 核心内容
**前端技术栈：**
```
React 18 + TypeScript + Vite
├── 状态管理：React Context
├── 画布引擎：tldraw (无限画布)
├── 本地存储：IndexedDB (idb)
├── 样式系统：Tailwind CSS
└── 国际化：i18next (中英双语)
```

**后端技术栈：**
```
Vercel Serverless Functions
├── AI对话：Replicate API (Claude 4.5 Sonnet)
├── 图像生成：RunComfy (Seedream 4.5)
├── 视频生成：RunComfy Video API
└── 图片托管：ImgBB CDN
```

**核心创新：**
1. **StaleTracker系统** - 智能数据过期追踪
2. **JSON Schema校验** - 确保AI输出质量
3. **流式响应处理** - 解决Vercel超时限制
4. **任务轮询机制** - 异步生成任务管理

**部署架构：**
```
全栈一体化部署在 Vercel
├── 前端：自动优化 + 全球CDN
├── API：Serverless Functions 自动扩缩容
└── 数据：IndexedDB本地 + 云端同步
```

## PPT设计建议

### 整体风格
- **配色方案**：
  - 主色：深紫渐变 (#667eea → #764ba2)
  - 辅色：青色 (#06b6d4)、粉色 (#ec4899)
  - 背景：深色系 (#0f172a)
  - 文字：白色/浅灰

- **字体选择**：
  - 标题：思源黑体 Bold / 苹方 Bold
  - 正文：思源黑体 Regular / 苹方 Regular
  - 代码：JetBrains Mono / Fira Code

- **视觉元素**：
  - 大量使用渐变色块
  - 图标化表达（避免大段文字）
  - 产品截图占重要位置
  - 数据可视化（图表、流程图）

### 每页设计原则
1. **一页一个核心信息** - 不要堆砌
2. **大标题 + 关键词** - 避免段落文字
3. **视觉优先** - 能用图表就不用文字
4. **配色统一** - 保持品牌色一致性
5. **留白充足** - 不要塞满每一寸空间

### 动画建议（适度使用）
- 封面：淡入效果
- 页面切换：推入/淡出
- 数据展示：计数动画
- 流程图：依次出现
- 产品演示：缩放+高亮
